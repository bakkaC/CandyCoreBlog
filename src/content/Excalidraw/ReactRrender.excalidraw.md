diff 是为了最小化更新，而diff 的前提是构造中的 element 有比较的对象

为其根节点添加 element的引用，防止垃圾回收，使用有 DOM 的引用的 element 对比是否可行？
事实上这个想法迭代后就是旧的Stack Reconciler的模型，这样就初步解决了 diff 的实现

element 和实例树的模型，没有一个中间层，存储临时状态
所以这种结构意味着必须边遍历边改DOM，因为递归栈无保存非当前节点的信息
所以修改必须立即落地，而这造成中间一旦报错，就会污染 DOM 结构，并无法回滚

所以我们希望把更新分为两个阶段，首先收集所有修改打上标记，再统一提交，防止错误导致的污染
能不能diff 时把“需要更新的 DOM”存进一个表，最后统一提交？

diff 修改过程会是一个不稳定的过程，上一个更新会影响下一个更新，
插入和删除会影响树的结构而树的结构是后续的插入删除依赖的内容，
所以似乎必须有一个边遍历边修改的结构，记录更新所依赖的状态

答案就是 fiber 的双缓冲结构，加入了一个 WIP的 草稿layer，来边遍历边修改，但是不立即修改 DOM
从 旧实例树+element+DOM 的三层，改为了旧 fiber+element+WIP fiber+DOM 的四层结构

fiber 从对象树改为链表树，遍历过程中修改指针，从而在内存结构中更新结构，标记修改flag，加入 effectList，
在构建整颗树结束之后再统一提交，修改DOM，诞生了现在 render phase 和 commit phase ^5oFPxcEo
