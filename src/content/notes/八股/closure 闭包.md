
## 定义

闭包（closure）本质是：**函数 + 它创建时所“记住”的外部作用域环境**。

函数的嵌套定义时：

嵌套定义时，允许用到父级的作用域中的变量，因为它包含了指向父级对象的指针
- 函数和数据包在了一起，称为闭包

创建函数时，frame stack会指向的对象，将函数对象和parentFrame的指针包在了一起

调用某个函数时，会创建一个栈帧，而栈帧中自然会带有对象里包装的指针，而有了这个指向父栈帧的指针，自然就能访问父帧栈的变量，就可以自然的实现一层层向上寻找变量的效果

![](assets/closure%20闭包/file-20260222225252343.png)

函数的定义所在的词法环境，决定了它的parent帧指向
函数调用时，会新开一个栈帧，将函数定义时创建的包入栈，

一个定义在全局环境的函数，被嵌套调用，它也不能指向其直接的父环境，而是跟随指针直接在Global中找变量

随定义链走，而不是随着调用链走 

parent的决定方式，是通过判断函数在哪个帧定义

## 闭包的作用

在 MDN Web Docs 的语境里，**closure（闭包）**不是“功能点”，而是一种由词法作用域带来的能力。它常见的实际用途主要有这些（从语言机制 → 编程效果的因果关系）：

**1）数据封装 / 私有变量**  
函数可以“记住”其创建时的作用域，从而让外部无法直接访问内部变量，只能通过暴露的函数间接操作。  
→ 在没有类私有字段的早期 JS 中，这是实现私有状态的核心手段。

**2）函数工厂（生成带配置的函数）**  
外层函数接收参数，内层函数在之后执行时仍能访问这些参数。  
→ 本质是把“配置”绑定进函数环境。

**3）保持状态（stateful function）**  
闭包能在多次调用之间保留变量值，而不需要全局变量。  
→ 常用于计数器、缓存、节流/防抖内部状态等。

**4）回调中的上下文保持**  
异步或事件回调执行时，原作用域通常已经结束；闭包让回调仍能访问当时的变量。  
→ 这是 JS 异步编程模型能自然工作的基础之一。

**5）模块模式（Module Pattern）**  
通过 IIFE + 闭包创建一个只暴露部分 API 的模块。  
→ 在 ES Module 普及前非常常见，本质仍是“封装 + 受控暴露”。

**6）函数式编程中的偏应用 / 柯里化基础**  
把部分参数固定在闭包里，返回等待剩余参数的函数。  
→ 你刚写的 `sum(x)(y)` 就是典型例子。



## 柯里化

```js
function sum(x) {
  let total = x;

  function adder(y) {
    total += y;
    return adder;   // ← 这里决定了还能继续调用
  }

  adder.valueOf = () => total;
  return adder;
}
```

1. 为了能持续调用，要返回函数对象
2. 要实现“多次调用共享一个 total”，就必须把变量放在外层作用域里
	1. 闭包的表现为创建了一个 total
	2. 调用结束了之后留下了 adder 对象
	3. 而 adder 对象用到了 total，对 sum 里创建的这个 total 还有被引用，所以 total 还没销毁
	4. 这使得链式调用 adder 的指针一层一层始终指向 total 对象帧
3. 那么返回的函数如何当做数字用呢，加上一句 valueof 行为的属性即可


你存储的其实是一个对象：

```js
adder = {  
  可调用行为: function(y) { total += y },  
  内部状态: total,  
  数值表示: valueOf() → total  
}
```

变量只是指向这个对象的引用，adder 对象遇到一些只有 number 类型能做的运算符，js 的运行时就会隐式的调用 valueof，返回 total
