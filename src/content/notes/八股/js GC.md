# **一、基本做法：标记清理（Mark-Sweep）**

传统 GC 流程：

1. 开始时认为所有对象都是“需要清理”的（白色）。
2. 从根对象出发遍历，把访问到的对象标记为 active（黑色）。
3. 遍历结束后，未标记的对象即为垃圾。
4. 执行清理。

**缺点**

标记清除算法有一个很大的缺点，就是在清除之后，剩余的对象内存位置是不变的，也会导致空闲内存空间是不连续的，出现了 `内存碎片`，并且由于剩余空闲内存不是一整块，它是由不同大小内存组成的内存列表，这就牵扯出了内存分配的问题

需要通过 **标记整理（Mark-Compact）算法** 进行内存整理

# **二、V8 的代际优化（Generational GC）**

V8 把堆划分成：

- **新生代（新对象）**
- **老生代（存活较久的对象）**


不同区域使用不同的回收算法，以获得最优性能。

---

# **三、新生代：Scavenge（复制算法 Copying GC）**

### **1. 新生代特点**

- 对象生命周期短，大部分对象在创建后很快死亡。
- 空间较小（几 MB）。
- 必须高频快速回收。

### **2. Scavenge 机制（From → To 复制）**

新生代堆分成两个等大小的半区：

- From 区（使用中）
- To 区（空闲）

GC 流程：

1. 扫描 From 区的对象。也是标记法
2. 将存活对象（active）复制到 To 区并排序。活对象都转移到空闲区（修改指针）
3. 执行引用更新，由于活对象指针修改了，from 区只剩下死对象，进行清空
4. From/To 角色互换。让真正空闲的区域变为空闲区，真正使用的区域变为使用区

### **3. 复制算法的优点**

- **只处理活对象，死对象自动被淘汰，不需要额外清理。**
- **没有内存碎片**（To 区是连续写入）。
- **分配速度快**（bump pointer 方式分配）。
- 非常适合新生代“死亡率高”的特性。

### **4. 晋升到老生代**

对象如果：

- 经历多次新生代 GC 仍然存活，或
- 占用空间较大

则会被晋升到老生代。

---

# **四、老生代：标记 + 清理 + 整理（Mark-Sweep + Mark-Compact）**

老生代对象多、存活时间长，不能用复制算法（成本太高）。

流程：

1. **标记（Mark）**  
    从根对象扫描，标记可达对象。

2. **清理（Sweep）**  
    回收未标记对象，但可能会产生碎片。

3. **整理（Compaction）**  
    将存活对象移动，避免碎片（仅在需要时做，不是每次都做）。


---

# **五、V8 老生代优化：增量标记**

老生代空间大，不能长时间阻塞 JS 线程，因此使用**增量标记**（Incremental Marking）：

- 把原本一次性做完的“标记”拆分，让 JS 和 GC 交替执行。
- 每次只处理一小部分标记任务。

### **三色法+写屏障：**

- **白色**：未访问（可能是垃圾）
- **灰色**：已访问，子节点未处理
- **黑色**：已访问，子节点也已处理

GC 持续把灰色处理成黑色，直到无灰色对象。
只有黑白二色，无法中断，因为一次执行的进度是没法表示的
引入灰色之后就有了进度的概念，知道一次标记是否完成，知道执行到哪里了

可以中断，可以增量的标记，使用户几乎感知不到 GC，pause time 大幅降低

在增量标记过程中 JS 仍在执行，可能会出现：

- 已标黑的对象新增指向白对象的引用
- 这会破坏三色不变式，造成漏标

解决方案就是写屏障

当 JS 代码执行，会引起 bug 的情况就是黑 → 白，意味着已扫描的对象更新，最终本该清理的对象不会被清理

写屏障使发生“黑 → 白”引用时，自动把白对象变成灰色（灰色一定会变为黑色，发生这个变色意味着这个节点就已经不会被删了，关键是其子节点），因为灰色把其子节点加入待处理队列了，不会漏扫

保证：
- 所有可达对象最终都会被标记到（变黑）
- 清扫时不会误删仍然可达的对象

---

# **七、增量标记为什么安全？**

- 增量标记阶段**不执行清扫**，不会删除对象。
- 最终会有一个短暂的 **Final Mark（最终标记）**：

    - 补齐所有可能漏掉的标记
    - 处理灰色队列

- Final Mark 之后，所有可达对象都为黑色；  
    白色对象才真正不可达，可执行清扫。


---

# **八、整体流程总结**

### **新生代**

- 算法：Scavenge（复制）
- 特点：速度极快、无碎片、适用于大量短命对象
- 晋升：多次存活的对象进入老生代

### **老生代**

- 算法：标记-清除 + 必要时压缩
- 优化：三色法、增量标记、写屏障
- 目标：减少 STW 时间（不阻塞 JS）
