## 方法论推测

或许牛马最重要的能力就是这些能力
牛马是真正做事的人，而不是拉项目找机会挣大钱博弈的人，所以做事的能力是最重要的
如何评估人做事情的能力？

发现问题之后，描述问题，分析问题，定位问题，对解决方案进行决策，最终落地解决问题的过程始终是最受重视的能力和思维，因为没有人会觉得套公式做题有什么难的，一个简单的问题往往不需要思考，但是一个复杂的问题往往无法避开上述的过程，所以你会什么技术不重要，技术有时间就能学，关键是你怎么落地上面提及的过程，并且展示它

展示这里也就是解释你解决问题的过程，就是结构化表达的重要性了，我认为对于没有这种能力的人是要有意识的进行训练的，因为我现在就深受此困，所以我在这里尝试会推测应该怎么做

基本原则是要对问题以及表达内容按照树状结构进行拆分，我先前的问题就是仅仅停留在把问题拆成几个零散的点上，对于结构只有很模糊的认识，缺乏结构的主线，只包含了我在意的细节

比较常见的定结构主线的方法是 STAR 法则，在此基础上，有意识的先对问题整体进行总结，即形成总分结构，应该能对问题本身讲的很清楚的

| 字母  | 含义        | 解释      |
| --- | --------- | ------- |
| S   | Situation | 情境 / 背景 |
| T   | Task      | 任务 / 目标 |
| A   | Action    | 你采取的行动  |
| R   | Result    | 结果      |

还有的想到的模式是思维链或者逻辑链，即联想式的链式的阐述

事实上复杂的系统都是网状的，我需要为了讲述找到一条主线而做简化，而我理解一个复杂事物的时候都是在收集离散的点，连接随机的线，当我感觉把点收集完了，我认为这件事情就结束了，这会带来讲述上的困难

有机会还是实践一下吧，也不知道自己随便复盘想的东西对不对
